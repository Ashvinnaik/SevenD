# Definition of Done (DoD)

> Exit criteria for each 7D phase. A phase is "done" only when ALL criteria are met AND the checkpoint is passed.
> Consolidated from issue templates and agent guidance.

---

## Define — Done

- [ ] Problem statement is written and specific
- [ ] Affected users/stakeholders are identified
- [ ] Success criteria are measurable (with numbers)
- [ ] Scope (in/out) is documented
- [ ] Constraints and assumptions are listed
- [ ] **CP-1 Checkpoint: Agent approved + Human approved**

---

## Design — Done

- [ ] Minimum 2 solution options considered with tradeoffs
- [ ] Recommended approach selected with clear rationale
- [ ] Technical architecture documented
- [ ] Key design decisions recorded (ADRs)
- [ ] Risks identified and flagged
- [ ] **CP-2 Checkpoint: Agent approved + Human approved**

---

## Discover — Done

- [ ] Key assumptions explicitly identified and tested
- [ ] Prototypes/spikes completed for risky areas
- [ ] Findings documented with evidence
- [ ] Design updated based on learnings
- [ ] Remaining risks are documented and acceptable
- [ ] **CP-3 Checkpoint: Agent approved + Human approved**

---

## Document — Done

- [ ] Implementation spec is complete and unambiguous
- [ ] All user stories have testable acceptance criteria (Given/When/Then)
- [ ] Edge cases and error scenarios are covered
- [ ] Dependencies are identified and scheduled
- [ ] A developer unfamiliar with context could implement from this doc
- [ ] **CP-4 Checkpoint: Agent approved + Human approved**

---

## Diagnose — Done

- [ ] All tasks created as separate issues
- [ ] No task larger than "L" (3-5 days); XL tasks broken down further
- [ ] Dependencies between tasks are mapped
- [ ] Implementation order is logical
- [ ] Effort estimates provided for all tasks
- [ ] Existing codebase patterns identified
- [ ] **CP-5 Checkpoint: Agent approved + Human approved**

---

## Develop — Done

- [ ] All tasks implemented
- [ ] All acceptance criteria met
- [ ] Tests written and passing
- [ ] Code review complete, feedback addressed
- [ ] No critical bugs open
- [ ] Documentation updated (API docs, README if needed)
- [ ] **CP-6 Checkpoint: Agent approved + Human approved**

---

## Deploy — Done

- [ ] Deployment to all target environments complete
- [ ] Smoke tests passing in production
- [ ] Monitoring shows healthy metrics
- [ ] No critical alerts triggered
- [ ] Release notes published
- [ ] Stakeholders notified
- [ ] Rollback plan documented and tested
- [ ] **CP-7 Checkpoint: Agent approved + Human approved**
