# Definition of Ready (DoR)

> Entry criteria for each 7D phase. An item is "ready" for a phase only when ALL criteria are met.
> These complement the checkpoint gates — checkpoints are exit criteria, DoR is entry criteria.

---

## Ready for Define

- [ ] Problem or opportunity is identified
- [ ] Requester is available for questions
- [ ] Not a duplicate of an existing issue
- [ ] Issue created using Feature or Bug template

---

## Ready for Design (requires CP-1: Problem Approved)

- [ ] CP-1 passed — both agent and human approved
- [ ] Problem statement is specific and measurable
- [ ] Success criteria are defined
- [ ] Affected users/stakeholders are identified
- [ ] Scope boundaries are clear

---

## Ready for Discover (requires CP-2: Solution Approved)

- [ ] CP-2 passed — both agent and human approved
- [ ] Design has identified assumptions that need validation
- [ ] Risky areas are flagged for prototyping/spikes
- [ ] Timebox for discovery is agreed upon
- [ ] Resources for research/prototyping are available

---

## Ready for Document (requires CP-3: Validated Approach)

- [ ] CP-3 passed — both agent and human approved
- [ ] Discovery findings are documented
- [ ] Design is updated based on findings
- [ ] Remaining risks are documented and accepted
- [ ] Path forward is clear

---

## Ready for Diagnose (requires CP-4: Spec Complete)

- [ ] CP-4 passed — both agent and human approved
- [ ] Implementation spec is complete and unambiguous
- [ ] Acceptance criteria are testable
- [ ] Dependencies are identified
- [ ] A developer unfamiliar with context could work from the spec

---

## Ready for Develop (requires CP-5: Tasks Approved)

- [ ] CP-5 passed — both agent and human approved
- [ ] Tasks are created as individual issues
- [ ] All tasks are sized S, M, or L (no XL)
- [ ] Dependencies between tasks are mapped
- [ ] Implementation order is defined
- [ ] Assignee is confirmed for each task

---

## Ready for Deploy (requires CP-6: Code Complete)

- [ ] CP-6 passed — both agent and human approved
- [ ] All tasks are implemented and merged
- [ ] All tests passing
- [ ] Code reviewed and approved
- [ ] No open critical bugs
- [ ] Release notes drafted
